# Skill Atlas — план

Простыми словами, без лишнего. Реализация не начата.

---

## 1. Зачем

Программа, которая знает всё про ваши инструменты и отвечает на четыре вопроса:

- Что у нас есть?
- Что каждая вещь делает?
- Что взять под конкретную задачу?
- Что появилось нового и что протухло?

Доступна с компьютера, с телефона, а также из ChatGPT и Claude Code.

Главное правило: **программа ничего не двигает и не переименовывает в Git**. Только читает и показывает.

---

## 2. Что реально есть (посмотрел 15.07.2026)

Зашёл на `git.example.com` и пересчитал.

**99 репозиториев, две группы:**

| Группа | Сколько | Что это | Устройство |
|---|---|---|---|
| `design-lib` | 74 | Наборы по брендам: airbnb, apple, bmw, ferrari, figma… | Одинаковые: `DESIGN.md` + `README.md` + `preview.svg`, ~35 КБ |
| `skills-lib` | 25 | Скиллы | Разные: от одного `SKILL.md` до проекта на Python на 51 файл |

Всё вместе — 6 МБ.

**Ключевой факт: один репозиторий = одна карточка.** Заглянул внутрь — внутри всегда одна вещь, а не несколько. Значит 99 репозиториев дают ~99 карточек, а потолок в 500 репозиториев — 500 карточек.

Это в десять раз меньше, чем я предполагал сначала, и снимает почти все сложности.

**Карточка** — страничка одного инструмента: название, что делает, чем полезен, где лежит, что ему нужно. Как карточка товара в магазине.

## 3. Решения

| Что | Решение |
|---|---|
| Где живёт | Linux-сервер / NAS. Видеокарты нет — и не нужна |
| Доступ с телефона | Cloudflare Tunnel + домен |
| Вход | Логин и пароль |
| Источник | **Только Gitea.** GitHub — потом, но место под него оставлено |
| Приватные репозитории | **Не сканируются никогда.** Жёсткое правило в коде, не настройка |
| AI | Через OpenRouter. Один ключ на всё |

### Почему исчезла вся возня с приватностью

Сканируем только открытые репозитории. Раз содержимое и так открыто — отдавать его в AI не страшнее, чем показывать в браузере. Поэтому выброшены: галочки «AI можно / нельзя» по репозиториям, локальные модели, Ollama, матрица прав.

Осталось одно правило безопасности: не читать файлы с паролями и ключами (`.env`, `*.pem`, `id_rsa`) — вдруг кто-то случайно такое закоммитил.

### Место под GitHub

Работа с репозиториями спрятана за одну «розетку» — общий набор команд: дай список, дай файлы, что изменилось. Сейчас в розетку воткнута Gitea. GitHub — это вторая вилка, которую можно вставить позже, ничего больше не трогая.

## 4. Из чего собрано

| Часть | Чем |
|---|---|
| Всё приложение | Один процесс на Python. Без Redis, без отдельных сервисов |
| База | Один файл SQLite. На 500 карточек — с огромным запасом |
| Поиск по словам | Встроенный в SQLite |
| Поиск по смыслу | Векторы рядом, в том же файле |
| Страницы | Отдаются с сервера. Без React и сборки |
| Телефон | Те же страницы + значок на экран |
| Запуск | Один контейнер + Cloudflare Tunnel |

### Модели

| Зачем | Что | Почему |
|---|---|---|
| Поиск по смыслу | Qwen3-Embedding, 1024 числа | Первое место среди мультиязычных, понимает и русский, и код |
| Описания карточек | Claude Haiku 4.5 | Нужен объём, а не глубина |
| Рекомендации | Claude Opus 4.8 | Ради этого всё и делается — тут нужна голова |

Всё через OpenRouter — один ключ, смена модели одной строчкой в настройках.

**Поиск по смыслу** — это когда вы пишете «вытащить таблицы из пдф», а находится инструмент `pdf-table-extractor`, хотя ни одно слово не совпало. Работает так: каждый инструмент превращается в список из 1024 чисел, отражающих смысл — как координаты на карте, где похожее лежит рядом. Ваш вопрос превращается в такие же числа, и ищутся ближайшие точки.

## 6. Что делаем, а что нет

**Делаем:**

- Подключение Gitea, только открытые репозитории
- Дерево репозитория как есть, без перекладываний
- Карточки: три уровня описания (коротко / нормально / технически), картинка-превью у design-lib
- Автотеги с указанием, откуда взялись и насколько уверены
- Ручные теги главнее автоматических. Удалили автотег — он не вернётся после следующего сканирования
- Свои разделы: складываете инструменты как удобно, Git не меняется
- Поиск по словам и по смыслу, по-русски и по-английски
- Рекомендация трёх вариантов с объяснением, почему эти и чем плохи остальные
- Честный ответ «подходящего нет», если его правда нет
- Что нового, что изменилось, что удалилось
- Список протухшего — не трогали больше года
- Веб + телефон
- Доступ из ChatGPT и Claude Code

**Не делаем пока:**

- GitHub (место оставлено)
- Склейку одинаковых инструментов из GitHub и Gitea — без GitHub она не нужна
- Создание папок, веток, pull request — только чтение
- Оценки качества и проверки
- Работу без интернета на телефоне

## 7. Этапы

Сроков в днях не называю — работаем сессиями. После каждого этапа показываю результат.

| # | Этап | Что видно в конце |
|---|---|---|
| 1 | Каркас, база, подключение Gitea | В базе 99 репозиториев, приватные пропущены |
| 2 | Карточки и описания | 99 карточек с текстами, у design-lib — превью |
| 3 | Теги и поиск | Русский запрос находит английский инструмент |
| 4 | Рекомендации | «Нужен PDF в Excel» → три варианта с объяснением |
| 5 | Веб и телефон | Открывается с телефона, ставится на экран |
| 6 | ChatGPT и Claude Code | Те же данные видны оттуда |
| 7 | Изменения и протухшее | После правок в Gitea обновляется только изменившееся |
| 8 | Источник и обновления | Карточка знает, откуда взялась, и ставит новую версию |
| 9 | Одна дверь для добавления | Ссылка, сайт, скриншот или рилс → карточка |

**Риск один:** `skills-lib` разношёрстные — от одного файла до проекта на 51 файл. Может понадобиться подгонка на этапе 2. Всё остальное предсказуемо.

### Что сделано за 8 и 9

Этап 8. У 76 карточек записан источник и честная отметка — слепок на момент
копирования. Без отметки «файлы разные» ничего не значит: то ли вышла новая
версия, то ли вы сами поправили. `skill-atlas upstream` сравнивает,
`skill-atlas update <карточка>` ставит новую версию. Обновляется только то,
что вы не трогали; на остальное — отказ с причиной.

Живьём запись пока не проверялась: все 76 карточек совпадают с источником, а
портить рабочий файл ради проверки нельзя. Отказы и чтение проверены на живой
Gitea, запись — на тестах.

Этап 9. `skill-atlas find <что угодно>` — ссылка на GitHub, страница сайта,
скриншот, рилс или просто слова. Показывает кандидатов со звёздами и
описанием; выбираете вы. Автоматически не тащим: название на слух и с картинки
распознаётся неточно.

Модель тут только читает и слушает. Адрес репозитория, который она называет,
всегда проверяется у GitHub — на живом рилсе она выдумала `skills/last-30-day`,
которого не существует. Просить её не врать можно, полагаться на это нельзя.
